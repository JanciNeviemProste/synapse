### Migrácia path

Odporúčaný postup pre v2:

1. **Prisma migrations** — aktuálne sa používa `prisma db push` (z setup guide), nie migrations. Pred v2: zaviesť `prisma migrate dev` + commitovať `prisma/migrations/` do gitu, lebo nové moduly pridajú veľa modelov.
2. **Multi-tenancy / klient model** — aktuálne `Lead`, `Booking` nemajú `clientId/userId`. Ak má systém začať obsluhovať viacerých klientov agentúry, treba zaviesť `Client` / `Tenant` model a foreign-key naprieč. Inak nový obsah ostane single-tenant.
3. **Auth refactor** — pred public-facing v2 modulmi (napr. content scheduling pre klientov) nahradiť `ADMIN_PASSWORD` proper auth (Clerk / Supabase Auth / NextAuth migrácia by však znamenala aj UI refactor z EJS).
4. **Queue infra** — brief zakázal Redis, ale `cloner.service.ts` a `coder.service.ts` púšťajú prácu cez `.catch()` na Promise. Pre v2 ad generation + content scheduling treba persistent queue (postačí Postgres-backed `pg-boss` ak chceš ostať bez Redis).
5. **Logging / observability** — pridať Sentry alebo aspoň file-based winston pre nové cron joby (aktuálne len NestJS `Logger` → stdout).
6. **Test suite** — `0` testov je riskantné pri pridávaní nových modulov. Pred v2 pridať aspoň smoke testy pre `AiService`, `LeadsService`, `BookingService` (NestJS Testing module).
7. **Frontend modernization** (voliteľné, skôr brand-side) — EJS + Tailwind CDN bude nudný. Pre v2 kde-tu zvažuj Next.js dashboard ako separátny package, alebo ostať pri EJS + HTMX (zostane to lean).
